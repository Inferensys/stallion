<objective>
Create the in-container control server (agent-control package) that wraps the Claude Agent SDK, streams events over WebSocket, and exposes a control HTTP API. Also create the Dockerfile for the container image and shared sandbox type definitions.

Purpose: This is the foundational component -- the thing that runs INSIDE the Docker container. It replaces the old MissionEngine direct-call pattern with a network-isolated server. The backend will talk to this control server via HTTP+WS instead of calling query() directly.

Output: A new `packages/agent-control/` package with HTTP server, SDK executor, WS event relay, file API, and a Dockerfile. Shared types in `@stallion/shared` for the sandbox contract.

**Create 4 source files for the in-container control server.**

**`packages/agent-control/src/event-relay.ts`** -- WebSocket server that broadcasts SDKEnvelope objects:
- Create a `WebSocketServer` (from `ws`) attached to the HTTP server
- Validate `x-control-token` header on WS upgrade against `CONTROL_AUTH_TOKEN` env var
- Export `createEventRelay(server: http.Server, authToken: string)` returning `{ broadcast(data: unknown): void, wss: WebSocketServer }`
- `broadcast()` serializes to JSON and sends to all connected clients with `readyState === OPEN`
- On connection: log client connected. On close: log client disconnected.

**`packages/agent-control/src/executor.ts`** -- SDK query() wrapper:
- Export `async function runSession(params: { prompt, sessionId, resumeSessionId?, abortSignal? }, onMessage: (envelope) => void): Promise<{ sessionId: string, totalCostUsd?: number }>`
- Read `WORKSPACE_DIR` from env (default `/workspace`)
- Call `query({ prompt, options })` with:
  - `systemPrompt: { type: "preset", preset: "claude_code" }`
  - `tools: { type: "preset", preset: "claude_code" }`
  - `cwd: WORKSPACE_DIR`
  - `permissionMode: "bypassPermissions"`, `allowDangerouslySkipPermissions: true`
  - `abortController` wrapping the provided `abortSignal`
  - If `resumeSessionId`: add `resume: resumeSessionId`
- For each SDK message: wrap in `SDKEnvelope` shape `{ id: nanoid(), sessionId, timestamp: Date.now(), msg }` and call `onMessage()`
- Capture `session_id` and `total_cost_usd` from `result` type messages
- Return `{ sessionId: capturedSessionId, totalCostUsd }`
- On error: emit a final error envelope before throwing

**`packages/agent-control/src/file-api.ts`** -- workspace file browsing:
- Export `handleFileList(workspaceDir: string): Promise<string[]>` -- recursively walk workspace, skip `.claude` and `node_modules`, return relative paths
- Export `handleFileRead(workspaceDir: string, filePath: string): Promise<{ content: string } | { error: string }>` -- path-traversal check (resolved path must start with workspaceDir), read file as utf-8
- Reuse the `walkDir` pattern from existing `mission-manager.ts`

**`packages/agent-control/src/index.ts`** -- HTTP server entry point:
- Read env vars: `CONTROL_AUTH_TOKEN`, `WORKSPACE_DIR` (default `/workspace`), `CONTROL_PORT` (default `3001`)
- Create `http.createServer` with request handler
- Auth middleware: all requests must have `x-control-token` header matching `CONTROL_AUTH_TOKEN`. Return 401 on mismatch.
- Routes:
  - `POST /start` -- parse JSON body as `StartSessionRequest`, create an `AbortController`, store it, call `runSession()` in a non-blocking `.then()/.catch()`. Broadcast each message via the event relay. On completion, broadcast `{ type: "session_completed", sessionId, sdkSessionId, totalCostUsd }`. On error, broadcast `{ type: "session_error", sessionId, error: message }`. Return 200 `{ ok: true, sessionId }` immediately.
  - `POST /message` -- for follow-up messages (SDK resume). Parse body `{ prompt, sessionId, resumeSessionId }`. Same pattern as /start but with `resumeSessionId` set.
  - `POST /abort` -- call stored `AbortController.abort()`. Return 200 `{}`.
  - `GET /status` -- return `{ status: "ready"|"busy", sessionId?, uptime: process.uptime() }` based on whether a session is active.
  - `GET /files` -- call `handleFileList(WORKSPACE_DIR)`, return JSON array.
  - `GET /files/read?path=<relpath>` -- call `handleFileRead(WORKSPACE_DIR, path)`, return content or 400 error.
  - 404 for anything else.
- Attach event relay WebSocket server to the HTTP server
- Listen on `CONTROL_PORT`, log "Control server ready on :PORT"
- Handle SIGTERM/SIGINT: abort any active session, close server gracefully

IMPORTANT: The control server does NOT import `mission-env.ts` or `buildProcessEnv()`. Inside the container, env vars are already set by Docker (the backend sets them when creating the container). The control server just calls `query()` which picks up env vars automatically via `process.env`. Do NOT pass `env:` option to `query()` -- let the SDK read process.env naturally.

```dockerfile
FROM node:22-slim

# Install common dev tools CC needs (per CONTEXT.md: git, curl, jq, python3)

    git \
    curl \

    python3 \

    iptables \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/*

# Install Claude Code CLI globally
# The SDK needs the CLI installed for query() to work
RUN npm install -g @anthropic-ai/claude-code@latest

# Create workspace directory
RUN mkdir -p /workspace

# Copy agent-control source and dependencies
WORKDIR /app

# Copy package files first for better layer caching
COPY package.json ./
# Install deps (tsx for running TypeScript, ws, nanoid, etc.)
RUN npm install --omit=dev && npm install tsx

# Copy source code
COPY src/ ./src/
COPY tsconfig.json ./

# Default env vars (overridden by backend at container create)
ENV CONTROL_PORT=3001
ENV WORKSPACE_DIR=/workspace

EXPOSE 3001

# Run with tsx (TypeScript execution)
CMD ["npx", "tsx", "src/index.ts"]
```
